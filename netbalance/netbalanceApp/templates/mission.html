What it does

Our project is a management system application that automates the deployment of web-based applications, allowing them to be dynamically scaled based on load. Our application includes a custom web GUI that provides a user-friendly interface for controlling all functionality. It requires little to no technical skill to install and can be used to host on local servers. The application also monitors server statistics in real-time.

The system is hosted on a Raspberry Pi device, which serves as the management plane for the entire system. This makes it fully modular and allows all management applications to be run and processed on the individual device. Nonetheless, we've designed our application to run seamlessly on any computer-system, irrespective of the operating system.

The application utilizes Kubernetes and its scaling/self-healing functions, including horizontal pod auto-scaling, service discovery, and load balancing. The goal is not to replicate Kubernetes but rather to simplify the installation, management, and usage of the architecture dynamically via the Python program.

With this program, users require minimal technical knowledge of how Kubernetes works, and the deployment is fully automated. Users can adjust the number of hosts via the GUI, making it easy to configure servers simplistically without the need to interact directly with Kubernetes.

How we built it

Our project is built on the Django web framework, which we utilized to create an advanced web application with features such as authentication, dynamic website components, and security protocols. We also leveraged the vast array of Python libraries available in the Django framework to execute our Ansible automation, and enhance the functionality of the application.

To ensure the aesthetics of our web application aligned with its functionality, we incorporated Bootstrap into our design process. The end result was a visually stunning and user-friendly interface.

When it came to choosing a database management system, we carefully considered our options and ultimately selected SQLite for its superior performance and compatibility with the Django web framework. We made an informed decision to opt for a relational database management system over non-relational ones such as MongoDB, given that we did not require the complexity of non-relational databases.
